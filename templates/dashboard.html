ðŸ“š How It Works

1. Password Hashing

When you register, your password is hashed using bcrypt. Hashing is a one-way function that converts your password into a fixed-length string. Even if someone accesses the database, they cannot reverse the hash to get your original password.

2. Salting

Salting adds random data to your password before hashing. Each password gets a unique salt, so even if two users have the same password, their hashes will be different. This prevents rainbow table attacks.

3. Password Verification

During login, bcrypt extracts the salt from the stored hash, applies it to your entered password, and compares the result. If they match, login is successful.

4. Audit Logging

Every login attempt (success or failure) is logged with username, status, and timestamp. This creates an audit trail for security compliance and helps detect unauthorized access attempts.
